Business concept

Event Analytics intended to increase self-awareness and become a useful daily tool to organize and reflect on daily routines.

Implementation concept

Event Analytics consist of three subsystems:

  • Operational subsystem

    allows to create event types, events themselves, tags, etc

  • View subsystem

    present events in a way interesting and useful for end-users to make a trivial analytics. For example, I can see how many days a year I was ill. Or How many days a year I had procrastination. As each day is colored with an event type color, it is trival and inspirable to see the number of days events occurred, or the length of events changed during a year? month, etc.

  • Analytical subsystem

    provides report based on statics methods, MI, etc that can provide a useful and inspirable information about ourselves.
